COMET Release Branch Validation
Verify CRD tickets against GitHub release branches · github.com/cg-ict · jira.capgroup.com
⚠ Important: This tool must be run via the included launcher to work.
Double-click start.bat — it starts a local proxy server using PowerShell (built into Windows, no installs needed).
The proxy forwards API calls to JIRA and GitHub, avoiding browser CORS restrictions.

J JIRA Token

JIRA → Profile → Personal Access Tokens

G GitHub Token

Needs repo scope + SSO configured for cg-ict

R Release Details

Click Load to fetch Fix Versions from JIRA project CRD
Auto-filled from version. Edit if branch name differs.

G CRD Repositories (github.com/cg-ict)

Two checks per repository:
1. vs Previous Release — release/{prev} → release/{current} — finds commits with tickets that have wrong/empty Fix Version (accidental inclusions)
2. vs project/comet — release/{current} → project/comet — finds tickets assigned to current release that are in project/comet but may be missing from the release branch
